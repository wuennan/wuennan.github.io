用户相关概念

Linux下我们为什么要创建用户？

  1. 系统上的每一个进程(运行的程序)，都需要一个特定的用户运行。
  2. 通常在公司是使用普通用户管理服务器，因为root权限过大，容易造成故障。

用户相关文件

/etc/passwd: 记录用户信息，但是不记录密码。

[root@localhost ~]# head -1 /etc/passwd
root:x:0:0:root:/root:/bin/bash

/etc/passwd以:为分隔符，共可分为7列。

第一列 第二列 第三列 第四列 第五列 第六列 第七列
root x 0 0 root /root /bin/bash
用户名 密码占位符 用户UID 组GID 注释信息 用户家目录 用户登陆shell

/etc/shadow: 密码信息。

[root@localhost ~]# head -1 /etc/shadow
root:$xxx6:19489:0:99999:7:2:66275:

/etc/shadow以:为分隔符，共可分为9列。

第一列 root 用户名
第二列 $xxx6 密码密文，!!表示无密码
第三列 19489 最近修改密码的日期，从1970年1月1日到现在过了多少天。
第四列 0 密码最少使用天数。0表示随时可以修改，如果是n那么就表示设置了新密码以后n天之内都不能再修改这个密码。
第五列 99999 密码最长使用天数。99999表示不强制在多少天内修改密码。
第六列 7 密码到期提醒。当账号的密码有效期限快要到的时候，系统会根据设定的数值发出警告信息提醒用户密码还有多少天就要过期了。
第七列 2 密码过期后系统会强制要求用户必须重新设置新的密码才可以继续登录使用，如果密码过期几天后到了这个宽限时间用户依然没有登录更改密码，那么这么密码就会失效。
第八列 66275 账号失效日期。从1970年1月1日起，过多少天后，账户失效。
第九列 保留字段

系统对用户的约定，在CentOS7系统之前，UID1-499用于系统用户，而500+则用于普通用户。

用户UID 系统中约定的含义
0 超级管理员，最高权限，有着极强的破坏能力
1~200 系统用户，用来运行系统自带的进程，默认已创建
201~999 系统用户，用来运行用户安装的程序，所以此类用户无需登陆系统
1000+ 普通用户，正常可以登陆系统的用户，权限比较小，能执行的任务有限

用户管理

创建用户

在用户创建的过程需要参考/etc/login.defs和/etc/default/useradd这两个文件。如果在创建用户时指定了参数,则会覆盖/etc/login.defs和/etc/default/useradd相关内容。

/etc/login.defs相关配置内容

[root@localhost ~]# egrep -v "^$|#" /etc/login.defs 
MAIL_DIR	/var/spool/mail      邮箱地址
PASS_MAX_DAYS	99999            密码最长使用天数
PASS_MIN_DAYS	0                密码最短使用天数
PASS_MIN_LEN	5                密码的长度
PASS_WARN_AGE	7                密码到期前7天警告
UID_MIN                  1000    UID从1000开始
UID_MAX                 60000    UID从6w结束
SYS_UID_MIN               201    系统用户UID从201开始
SYS_UID_MAX               999    系统用户UID最大到999
GID_MIN                  1000    GID从1000开始
GID_MAX                 60000    GID最大到6w结束
SYS_GID_MIN               201    系统GID从201开始
SYS_GID_MAX               999    系统GID醉倒到999结束
CREATE_HOME	yes                  给用户创建家目录
UMASK           077
USERGROUPS_ENAB yes
ENCRYPT_METHOD SHA512

/etc/default/useradd相关配置内容

[root@localhost ~]# egrep -v "^$|#" /etc/default/useradd
GROUP=100
HOME=/home               用户默认的家目录
INACTIVE=-1              用户不失效
EXPIRE=                  过期时间
SHELL=/bin/bash          默认登陆shell
SKEL=/etc/skel           默认用户拷贝的环境变量
CREATE_MAIL_SPOOL=yes    创建邮箱

useradd命令用于新增用户，常用选项及说明如下表

选项 解释
-u 指定用户的UID
-g 指定用户的默认组，只能有一个
-G 指定用户的附加组，逗号隔开可以添加多个
-d 指定要创建的用户的家目录
-s 指定要创建用户的bash shell
-c 指定要创建用户的注释信息
-M 给创建的用户不创建家目录
-r 创建系统账户，默认无家目录

创建ennan用户，UID5001,基本组ops，附加组dev 注释信息:20220523,登陆shell:/bin/bash

用groupadd添加相关的用户组

[root@localhost ~]# groupadd ops
[root@localhost ~]# groupadd dev

创建用户并验证

[root@localhost ~]# useradd ennan -u 5001 -g ops -G dev -c "20220523" -s /bin/bash
[root@localhost ~]# id ennan
uid=5001(ennan) gid=1000(ops) groups=1000(ops),1001(dev)
[root@localhost ~]# grep "ennan" /etc/passwd
ennan:x:5001:1000:20220523:/home/ennan:/bin/bash

创建mysql系统用户，不建立用户家目录，并指定nologin使用户无法登陆系统。

# 使用-r可创建系统用户，默认不创建家目录
[root@localhost ~]# useradd mysql -r -s /sbin/nologin
[root@localhost ~]# id mysql
uid=995(mysql) gid=995(mysql) groups=995(mysql)
[root@localhost ~]# grep "mysql" /etc/passwd
mysql:x:995:995::/home/mysql:/sbin/nologin

修改用户

usermod命令用于更改用户信息，常用选项及说明如下表：

选项 解释
-a 更改用户组的时候，保存原有组不变，直接追加
-m 将家目录内容移至新位置 (仅于 -d 一起使用)
-l 指定要修改的用户的登录名
-u 指定用户的UID
-g 指定用户的默认组，只能有一个
-G 指定用户的附加组，逗号隔开可以添加多个
-d 指定要创建的用户的家目录
-s 指定要创建用户的bash shell
-c 指定要创建用户的注释信息

将用户ennan的UID改为9999，并添加附属组dba

查看用户ennan的信息

[root@localhost ~]# id ennan
uid=5001(ennan) gid=1000(ops) groups=1000(ops),1001(dev)

添加用户组dba

[root@localhost ~]# groupadd dba
# 若不加-a选项，附属组不会保留原先设置的组，会直接变更为dba
[root@localhost ~]# usermod ennan -u 9999 -aG dba
[root@localhost ~]# id ennan
uid=9999(ennan) gid=1000(ops) groups=1000(ops),1001(dev),1002(dba)

将用户ennan的用户名修改为wu，家目录修改问/tmp/wu

更改用户家目录时应加上-m选项，会将用户家目录移动到/tmp目录

[root@localhost ~]# usermod ennan -md /tmp/wu -l wu
[root@localhost ~]# id wu
uid=9999(wu) gid=1000(ops) groups=1000(ops),1001(dev),1002(dba)
# 更改用户家目录加了-m选项
[root@localhost ~]# su - wu
Last login: Mon Aug  5 15:43:49 CST 2019 on pts/1
[wu@localhost ~]$

更改用户家目录，没加-m选项，因为没有移动之前的家目录到/tmp/test目录，会提示家目录不存在。

[root@localhost ~]# usermod wu -d /tmp/test
[root@localhost ~]# su - wu
Last login: Mon Aug  5 15:45:28 CST 2019 on pts/1
su: warning: cannot change directory to /tmp/test: No such file or directory
-bash-4.2$

删除用户

userdel命令用于删除用户，常用选项及说明如下表：

选项 解释
-r 删除用户的同时删除他的家目录

删除用户user1，不删除用户家目录等其他目录。

[root@localhost ~]# userdel wu

删除用户user1及其家目录

[root@localhost ~]# userdel -r wu

用户数据可能会放置于家目录内，-r选项删除用户时，会删除其家目录，慎用！！！

其他用户相关操作

who命令

who命令用于查看目前登录系统的用户信息。

[root@localhost ~]# who
root     tty1         2019-08-05 08:26
root     pts/0        2019-08-05 08:27 (192.168.16.44)

whoami命令

whoami用于查看当前登陆系统的用户是谁

[root@localhost ~]# whoami
root

w命令

w命令用于显示已经登陆系统的用户列表，并显示用户正在执行的指令。

[root@localhost ~]# w
 16:31:29 up  1 day,  3 users,  load average: 0.00, 0.01, 0.05
USER     TTY      FROM             LOGIN@   IDLE   JCPU   PCPU WHAT
root     tty1                      08:26    8:04m  0.05s  0.05s -bash
root     pts/0    192.168.16.44    08:27    3:01m  0.92s  0.92s -bash
root     pts/1    192.168.16.44    14:42    1.00s  0.90s  0.03s w

说明

显示信息 说明
16:31:29 当前系统时间。
up 当前系统运行的时长。
3 users 有多少用户登陆了系统。
load average 当前系统负载状态。
0.00 1分钟内平均负载。
0.01 5分钟内平均负载。
0.05 15分钟内平均负载
JCPU 该tty终端连接的所由进程占用的时间，不包括过去的后台作业时间。
PCPU 当前进程(即w项中显示的)所占用的时间。

密码管理

为新用户添加密码

只有root才能执行才能为新用户添加密码。添加密码非为交互式和非交互式两种。

  • 交互式添加密码。

    [root@localhost ~]# useradd ennan   
[root@localhost ~]# passwd ennan
Changing password for user ennan.
New password: 输入密码
BAD PASSWORD: The password is a palindrome
Retype new password: 确认密码
passwd: all authentication tokens updated successfully.

  • 非交互式添加密码。

    [root@localhost ~]# echo "qwert" | passwd --stdin ennan
Changing password for user ennan.
passwd: all authentication tokens updated successfully.

    通过脚本，批量添加用户

    [root@localhost ~]# cat user.sh 
for i in {1..100}
do
	useradd test$i
	echo "123456" | passwd --stdin test$i
done

变更密码

为别人修改密码（只用root才能执行）

[root@localhost ~]# passwd test50
Changing password for user test50.
New password: 
BAD PASSWORD: The password is a palindrome
Retype new password: 
passwd: all authentication tokens updated successfully.

为自己修改密码（自己就可以）

[test50@localhost ~]$ passwd

New password:

生成随机密码

通过$RANDOM随机生成密码

$RANDOM生成随机数，将随机数通过md5加密，取加密后的6-15字符。

[ennan@localhost ~]$ echo $RANDOM | md5sum | cut -c 6-15
45de81b10c

通过mkpasswd随机生成密码

[root@localhost ~]#  yum install expect -y
# 安装扩展包
[root@localhost ~]# mkpasswd
7q~HfjM0g
[root@localhost ~]# mkpasswd -l 10 -d 3 -c 2 -C 3 -s 2
# -l 密码长度  -d 数字个数  -c 小写字母个数  -C 大写字母个数  -s 特殊字符个数
6u36~LRSo&

用户组管理

创建用户组 创建GID为666的用户组g1

[root@localhost ~]# groupadd -g 666 g1
[root@localhost ~]# grep "666" /etc/group
g1:x:666:

创建系统用户组g2

[root@localhost ~]# groupadd -r g2
[root@localhost ~]# grep "g2" /etc/group
g2:x:665:

修改用户组 将g1的GID改为777

[root@localhost ~]# groupmod -g 777 g1
[root@localhost ~]# grep "g1" /etc/group
g1:x:777:

将g1的用户名修改为gg

[root@localhost ~]# groupmod g1 -n gg
[root@localhost ~]# grep "777" /etc/group
gg:x:777:

删除用户组

可通过groupdel命令删除用户组，若用户组里有用户的话，需要先删除组内的用户才可以删除该组。

[root@localhost ~]# groupdel g1

用户提权

su

su命令用于切换到其他用户，除root外，使用改命令需要输入密码。

切换用户不改变原用户的工作目录，及其他环境变量目录。

[ennan@localhost ~]$ su root
Password:
[root@localhost ennan]#

切换用户后，同时切换到新用户的工作环境中，加载全部的环境变量。

[ennan@localhost ~]$ su - root
Password:
Last login: Thu May 25 18:22:34 CST 2023 on pts/0
[root@localhost ~]#

使用指定用户执行命令，这里以使用root用户执行命令为例。

[ennan@localhost ~]$ ll /root/.ssh
ls: cannot access /root/.ssh: Permission denied

[ennan@localhost ~]$ su root -c "ls -l /root/.ssh"
Password:
total 4
-rw------- 1 root root 575 May 25 16:17 authorized_keys

su切换用户的缺点：需要知道用户名对应的密码，很不安全。

sudo

用户通过sudo命令，可调用指定的命令。

使用sudo命令

普通用户执行yum命令，不能成功。

[ennan@localhost ~]$ yum install wget
Loaded plugins: fastestmirror
You need to be root to perform this command.

使用sudo命令提权，提示用户不在sudoers文件中。

[ennan@localhost ~]$ sudo yum install wget

We trust you have received the usual lecture from the local System
Administrator. It usually boils down to these three things:

    #1) Respect the privacy of others.
    #2) Think before you type.
    #3) With great power comes great responsibility.

[sudo] password for ennan: 输入密码
ennan is not in the sudoers file.  This incident will be reported.

通过root用户执行visudo命令，对用户进行权限设置。

# 查看yum命令路径
[root@localhost ~]# whereis yum
yum: /usr/bin/yum /etc/yum /etc/yum.conf /usr/share/man/man8/yum.8
# 修改配置
[root@localhost ~]# visudo 
# 添加
ennan ALL=(ALL) /usr/bin/yum
# 检查文件
[root@localhost ~]# visudo -c
/etc/sudoers: parsed OK

登陆到ennan用户，检查用户的权限

[ennan@localhost ~]$ sudo -l
[sudo] password for ennan: 输入密码

User ennan may run the following commands on localhost:
    (ALL) /usr/bin/yum

利用sudo执行yum命令，可以成功执行。

[ennan@localhost ~]$ sudo yum install wget
Loaded plugins: fastestmirror
Loading mirror speeds from cached hostfile
Package wget-1.14-18.el7_6.1.x86_64 already installed and latest version
Nothing to do

sudo别名提权

可使用sudo中自带的别名操作, 将多个用户定义成一个组。针对用户组进行提权。 用root用户执行visudo命令，对用户进行权限设置。

[root@localhost ~]# visudo

添加如下内容设置用户组别名及命令别名

root    ALL=(ALL)       ALL
# 设置组别名，test2和test3为OPS组；test4为DEV组。
User_Alias OPS = test2,test3
User_Alias DEV = test4
# 设置命令别名，NETWORKING为ifconfig和ping的别名
Cmnd_Alias NETWORKING = /sbin/ifconfig, /bin/ping
Cmnd_Alias SOFTWARE = /bin/rpm, /usr/bin/yum
Cmnd_Alias STORAGE = /bin/mount, /bin/umount
# 将组的别名和命令别名对应起来
OPS ALL=(ALL) NETWORKING,SOFTWARE,STORAGE
DEV ALL=(ALL) NETWORKING

检查配置

[root@localhost ~]# visudo -c
/etc/sudoers: parsed OK

用test3用户登陆系统，查看权限

[test3@localhost ~]$ sudo -l

    .......
    .......

User test3 may run the following commands on localhost:
    (ALL) /sbin/ifconfig, /bin/ping, /bin/rpm, /usr/bin/yum,
        /bin/mount, /bin/umount

用test4用户登陆系统，查看权限

[test4@localhost ~]$ sudo -l

    .......
    .......

User test4 may run the following commands on localhost:
    (ALL) /sbin/ifconfig, /bin/ping

sudo用户组提权

sudo可针对用户组进行提权，可使用groupadd将用户添加到用户组，然后给组分配sudo的权限。如果有新 用户加入,直接将用户添加到该组。 将用户添加到对应的组中

[root@localhost ~]# groupadd group_dev
[root@localhost ~]# groupadd group_op
[root@localhost ~]# usermod test5 -G group_dev 
[root@localhost ~]# usermod test6 -G group_dev 
[root@localhost ~]# usermod test7 -G group_op
[root@localhost ~]# id test5
uid=3010(test5) gid=3010(test5) groups=3010(test5),6674(group_dev)
[root@localhost ~]# id test6
uid=3011(test6) gid=3011(test6) groups=3011(test6),6674(group_dev)
[root@localhost ~]# id test7
uid=3012(test7) gid=3012(test7) groups=3012(test7),6675(group_op)

修改配置文件

[root@localhost ~]# visudo

visudo主要添加了以下的内容

Cmnd_Alias NETWORKING = /sbin/ifconfig, /bin/ping
Cmnd_Alias SOFTWARE = /bin/rpm, /usr/bin/yum
Cmnd_Alias STORAGE = /bin/mount, /bin/umount
# %代表组，如果不加%，系统会认为是一个用户名
%group_dev ALL=(ALL) NETWORKING,STORAGE
%group_op ALL=(ALL) SOFTWARE
